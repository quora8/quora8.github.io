Google Core Web Vitals 优化：初学者指南

SEO
|阅读量:

Core Web Vitals 是一组与网站速度、视觉稳定性和交互性相关的指标。它是将于 2021 年 6 月中旬推出的页面体验更新的一部分。

如果您是网站所有者，则需要立即采取措施优化您的网站以使用 Core Web Vitals，以确保您的自然排名安全。

在本文中，我们将了解什么是 Core Web Vitals 以及提高网站性能的基本步骤。

什么是核心网络生命力？
Core Web Vitals 是一组用于提高网站性能的指标。当前的 Core Web Vitals 指标包括三个方面：加载、交互性和视觉稳定性。

Core Web Vitals 帮助网站管理员专注于对他们的商业网站最重要的指标。这些指标直接提高了网站的性能，并有助于提供卓越的用户体验。

核心 Web Vitals 的主要因素是什么？

2021 Core Web Vitals 测量中包含的三个主要因素是：

  1. LCP (Largest Contentful Paint) - 它测量浏览器在您的网站上加载最大元素所花费的时间。理想的 LCP 应小于 2.5 秒。超过 4 秒被认为是差的。

  2. FID（首次输入延迟） - 它衡量访问者在您的网站上采取任何行动所花费的时间。花费的时间越长，FID 得分就越高。理想的 FID 值应小于 100 ms。超过 300 ms 被认为是较差的。

  3. CLS（累积布局偏移） - 它测量您的网页在加载时所做的布局偏移。理想的 CLS 应小于 0.1。高于 0.25 被认为是差。

优化核心 Web Vitals 所需的步骤
以下是针对 Core Web Vitals 优化您的网站的步骤：

第 1 步：登录 Google 搜索控制台

选择您要修复其核心网络生命力的属性。单击“体验”选项卡下的“Core Web Vitals”部分。您会看到网站上不良 URL 的数量以红色标记。

第 2 步：在 Core Web Vitals 报告中单击 Open Report

在这里，您可以看到影响您网站的确切 Core Web Vitals 问题。单击问题以查看有问题的示例 URL。

第 3 步：分析 URL

获得未通过 Core Web Vitals 评估的 URL 后，请转到Page Speed Insights 工具生成解决问题的建议。

在 Page Speed Insights 工具中输入示例 URL，然后单击分析。

现在，您可以查看准确的 LCP、FID 和 CLS 值以及一组帮助您解决问题的建议。

或者，您也可以使用Finteza Lighthouse 报告。登录到 Finteza 仪表板并单击 Lighthouse。在这里，您可以查看网站的整体性能得分以及可访问性得分和 SEO 得分。

性能指标可让您衡量六个关键指标，即首次内容绘制、首次有意义绘制、速度指数、首次 CPU 空闲、交互时间和估计输入延迟。

该报告还提供了改进提示，以修复所有关键指标并提高您的 Core Web Vitals 分数。

第 4 步：验证修复

一旦您按照 Page Speed Insights 和 Finteza 工具的建议在您的网站中实施了所有更改，就该验证修复了。

您应该访问 Search Console 中的 Core Web Vitals 报告，然后点击 Validate Fix。

Google 将需要大约 28 天的时间来审核修复程序，因此您需要耐心等待审核过程。

您需要做哪些改变？
以下是您可以在您的网站中实施的一些即时更改，以解决一些 Core Web Vitals 问题：

将您的网站托管升级到更快、更大的带宽。
优化您网站上的所有图像。
对图像使用 WebP 格式，因为它比 PNG 快 32 倍。
尽量减少网站上不必要的插件数量，因为它们会提高加载速度。
使用 CDN 并通过有效的缓存策略提供静态资产。
删除未使用的 JavaScript。
确保在 webfont 加载期间文本保持可见。
消除渲染阻塞资源并保持低请求数。
缩小 CSS 并删除未使用的 CSS。
预加载密钥请求并避免不必要的重定向。
下一步是什么？

Google 将在 6 月中旬推出页面体验更新，因此您只有几周的时间来优化您的网站以适应 Core Web Vitals。请记住，每个网站所有者都应在其网站上进行更改以通过 Core Web Vitals 评估。尽早开始以确保您的自然排名安全。

提高 Google Core Web Vitals 中 CLS 分数的 8 种方法

Core Web Vitals 是谷歌去年发布的一组网站用户体验指标——以及即将成为排名的因素。这些新指标评估谷歌所谓的“页面体验”并相应地提供分数。

Core Web Vitals 涵盖了页面体验的三个方面：最大内容绘制 (LCP)、首次输入延迟 (FID) 和累积布局偏移 (CLS)。

虽然 LCP 和 FID 指标侧重于使用户感觉“快速”的网站体验的不同方面，但 CLS 侧重于网页加载时发生的内容“移动”量。如果您曾经尝试在页面加载时单击按钮、链接或图像，但由于元素不停地跳来跳去而无法单击，那就是布局转换，这是一种糟糕的体验。

尽管在 Google 计算 Core Web Vitals 分数时，这三个方面（LCP、FID 和 CLS）的权重相同，但有时提高网站的 CLS 分数可能是最困难的。

什么是累积布局移位 (CLS) 的示例？

让我们直奔源头。谷歌这样解释累积布局转换：

“你有没有在网上阅读文章时页面突然发生变化？没有任何警告，文本移动了，你已经失去了自己的位置。或者更糟糕的是：你正要点击一个链接或一个按钮，但在你的手指落地前的瞬间——砰——链接移动了，你最终点击了别的东西！”
很高兴这个长期存在的网络问题终于得到了谷歌的大量关注，很明显该公司认真对待它。

在 Google 的Lighthouse测试工具中，CLS 对总分的影响相对较小（占总分的 5%）。然而，在 Core Web Vitals 中，CLS 贡献了总分的三分之一。当您记得 Core Web Vitals 分数很快将作为排名因素包含在 Google 算法中时，这一点就显得尤为重要。

提高网站 CLS 分数的提示

一些网站构建器平台将通过自动执行许多优化来为您改进 CLS，以便与最佳实践保持一致，从而完成大量所需的繁重工作。但是，如果您不使用这些平台之一，则需要手动确保您的网站符合 Google 的新指南。

无论您是否与网站建设者合作，熟悉以下最佳实践仍然是有益的。

  • 注意图像的宽度和高度属性：
    确保图像和视频具有明确指定的宽度和高度属性。这告诉浏览器在页面开始加载时在布局中为这些元素保留多少空间。
  • 小心使用自定义字体：
    如果您必须使用自定义字体，请尽可能将它们直接嵌入 HTML。
    当访问者登陆网页时，通常会首先加载后备字体。
    然后，当下载自定义字体时，会将其加载到页面中。
    加载该自定义字体后，可能会发生布局变化，因为自定义字体的大小与已显示的备用字体的大小不同。如果您在 HTML 响应中嵌入自定义字体，这将确保在初始页面加载时下载它。（注意：必须谨慎使用此方法，因为您不想通过内联过多数据来使初始 HTML 响应膨胀，这可能会对其他页面体验指标产生负面影响。）
  • 对动画使用 transform CSS 属性：
    在实现动画时——尤其是基于滚动的动画——一定要使用 transform CSS 属性而不是其他方法。此外，您希望避免移动动画元素的高度或宽度，除非它是在单击事件之后（例如，展开手风琴小部件）。
  • 为广告、嵌入和 iframe 节省空间：
    在页面上放置广告（或其他第三方内容）时，请确保为该元素提供足够的空间以在页面模板中加载。这意味着为将加载广告的区域指定高度和宽度。与图像类似，这通常意味着将宽度和高度属性直接应用于元素。
  • 避免在首屏使用动态大小的内容：
    有时在加载之前无法知道元素的高度。例如，来自 Disqus 或 Facebook 的评论框可能缺少您可以设置的特定高度属性。在这些情况下，这些元素应该在页面上移动到更低的位置——避免将它们放在折叠上方的初始视口中。由于 CLS 计算的重点是何时移动视口中的元素，因此在主视口下方加载动态大小的内容将减少 CLS 影响。
    不要在没有用户交互的情况下动态加载内容：如果您加载电子邮件注册表单、广告或其他任何内容，它会改变页面，这意味着它会直接导致布局偏移。但是，如果新元素出现在单击后 500 毫秒内，您可以在单击后加载它们。
  • 不要在现有内容上方加载新内容：
    如果您向页面添加新内容（例如，页面顶部的广告横幅更新），这将移动其下方的所有内容。如果用户已经在页面上向下滚动，上面的内容将会移动，用户将失去他们在页面上的位置。应不惜一切代价避免这些行为。
  • 避免无限滚动更改滚动位置上方的内容：
    加载滚动下方的内容很好，但更改滚动上方的内容可能会触发绘制事件（即，浏览器重新计算页面上所有内容的位置）。

结论

每次 Google 提出新的网站测试或评分方法时——从移动友好性测试到 Lighthouse，再到现在的 Core Web Vitals——数字营销人员、开发人员和网络专业人士都会疯狂地了解最新的基准指标和使网站与 Google 最新推荐的最佳做法保持一致。

果然，这家搜索巨头去年发布了Core Web Vitals，随后宣布这些指标将在 2021 年 5 月添加到其排名算法中，再次让网络在某些圈子中陷入了轻微的恐慌。

事实上，根据Duda 最近的一项研究，“大约 92% 的网络专业人士认为这些更新最终会对他们网站的搜索排名产生中度至显着的影响。” 然而，同一项研究还发现，超过一半的受访者表示“他们的大多数网站还没有准备好接受任何强调核心 Web Vitals 重要性的谷歌算法变化。”

希望凭借我们在本文中介绍的信息，您不会成为他们中的一员。通过遵循上述方法，您应该会发现自己处于有利位置，可以为您管理的任何网站准备好 CLS 分数和 Google 即将进行的算法更改。
